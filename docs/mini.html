safaribooks

Async O'Reilly book downloader — cookies, retries, rate limits, EPUB output

⚠ The Problem

O'Reilly killed programmatic login. The v1 API is dead. The upstream tool is abandoned. Your reading list is trapped behind a browser window.

⚡ The Solution

Cookie-based async downloader with intelligent retries, rate limiting, and sequential asset fetching. Extracts cookies from your browser session, downloads every chapter, image, and stylesheet, and packages them into a valid EPUB.

✔ The Payoff

A complete, offline-readable EPUB in minutes. No more browser tabs. No more expired sessions killing downloads halfway through.

▶ Download Pipeline

The 14-step pipeline from authentication to final EPUB. Steps 7–10 run sequentially (CSS → fonts → images → videos).

  1. 1
    check_login()→ ApiClient
    GET /profile/ — validates session cookies are active
  2. 2
    fetch_book_info()→ BookInfo
    GET /api/v2/epubs/urn:orm:book:{id}/ — title, isbn, chapters, cover
  3. 3
    enrich_metadata()→ SearchAPI
    GET /api/v2/search/ — enriches authors, publishers, description. Skips on error.
  4. 4
    ensure_book_dirs()→ BookPaths
    Creates OEBPS/, Styles/, Images/, Video/, META-INF/
  5. 5
    fetch_chapters()→ Chapter[]
    Resolves chapter list with content_urls. Typically 20–40 chapters.
  6. 6
    process_chapters()→ ParseResult[]
    For each chapter: fetch HTML → parse → rewrite links → write XHTML
  1. 7
    download_css()
    Style00.css, Style01.css, ...
  2. 8
    download_fonts()
    Scans CSS for @font-face urls. Formats: otf, woff2, ttf
  3. 9
    download_images()
    jpg, png, gif, svg. Optional max_width resize.
  4. 10
    download_videos()
    mp4 files. Optional — many books have none.
  1. 11
    render_content_opf()
    Generates OEBPS/content.opf — manifest, spine, guide, dc:metadata
  2. 12
    render_toc_ncx()
    Generates OEBPS/toc.ncx — NCX navMap from table-of-contents API
  3. 13
    build_epub()→ EPUB
    zipfile.ZipFile: mimetype STORED first, rest DEFLATED
  4. 14
    copy_to_library()
    Copy epub to library dir + save cookies snapshot

🔑 Cookie Auth Flow

Sequential swimlane flow showing cookie setup and runtime authentication.

Setup Flow
  1. User → CLI safari auth setup
  2. CLI → CookieManager auto-detect format (JSON, Netscape, browser-cookie3)
  3. CookieManager ↻ CookieManager validate cookies (groot_sessionid, jwt, csrf, logged_in)
  4. CookieManager → CLI save to ~/.config/safaribooks/cookies.json
  5. CLI → User Auth configured!
Runtime Flow
  1. User → CLI safari fetch 9781491958698
  2. CLI → ApiClient load cookies from disk
  3. ApiClient → O'Reilly GET /profile/
  4. O'Reilly → ApiClient 200 OK ✓
  5. O'Reilly → ApiClient 401/403 ✗ — triggers cookie refresh
  6. ApiClient → CookieManager _try_cookie_refresh() — reload cookies.json, retry

Backoff formula: base_delay(1s) × 2^attempt × (1 ± 0.25 jitter), capped at 60s. For 429/503 with Retry-After header: header_value × 2.0.
Token Bucket Rate Limiter
Token Bucket
2 / 2 tokens
Refill: 1.0 tokens/s  |  Burst: 2 max  |  Guard: asyncio.Lock
Requests consume 1 token. If tokens = 0, callers await until refill. The async Lock prevents race conditions between concurrent coroutines.

§ Chapter Processing

Each chapter flows through a six-stage transformation pipeline before being written as XHTML.

🌐 fetch_html GET chapter content_url from O'Reilly CDN. Returns raw HTML.
🌳 lxml_parse Parse HTML with lxml.html. Build element tree for traversal.
🔍 extract_assets Walk tree for <img>, <link>, <video> URLs. Collect CSS, images, videos.
🔗 rewrite_links Replace absolute CDN URLs with relative EPUB paths (Images/, Styles/).
🖼 detect_cover First chapter, first <img> — extract as default_cover.jpeg if no metadata cover.
📄 serialize_xhtml lxml.html.tostring() with XHTML doctype. Write OEBPS/{filename}.xhtml.
